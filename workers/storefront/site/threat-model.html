我们重点防护的风险类型

Risks We Focus On

1. 间接提示词注入

通过网页、文档或消息中的隐藏指令，诱导 AI Agent 执行非预期操作。

1. Indirect Prompt Injection

Hidden instructions in web pages, documents, or messages trick AI agents into performing unintended actions.

2. 恶意 Skills / MCP 服务器

供应链投毒：看似正常的扩展内含窃取密钥、植入后门等恶意行为。

2. Malicious Skills / MCP Servers

Supply chain poisoning: seemingly normal extensions that steal keys, plant backdoors, or exfiltrate data.

3. 过度授权

Agent 拥有超出实际需要的权限，一旦被利用后果会被放大。

3. Over-Permissioning

Agents granted unnecessary privileges, amplifying damage if exploited.

4. 明文密钥暴露

API 密钥以明文存放在配置文件中，任何能读取文件的进程都可获取。

4. Plaintext Key Exposure

API keys stored in plaintext config files, accessible to any process that can read those files.

5. 未知出站连接

Agent 连接到未经批准的外部服务，可能泄露敏感数据。

5. Unknown Outbound Connections

Agents connecting to unapproved external services, potentially leaking sensitive data.

6. 配置漂移

权限被悄悄扩大，用户未收到通知或确认。

6. Config Drift

Permissions silently expanded without user notification or confirmation.

7. 宿主环境篡改

未经授权修改宿主工具配置，改变工具行为或注入恶意设置。

7. Host Environment Modification

Unauthorized changes to host tool configurations, altering behavior or injecting malicious settings.
