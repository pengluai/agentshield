# Creem 团队审核待通过时的对接状态（2026-03-18）

## 当前状态结论
基于 2026-03-18 用户提供的最新截图：
- 默认 payout method 已设为 `USDC Wallet`
- `Add an account to get paid` 已完成
- 但右侧 checklist 仍停在：`Review by Creem Team`
- `Payouts Enabled` 仍未完成

因此，当前结论是：
**店铺仍未正式进入 live 收款状态。**

## 实际支付可用性验证
对当前项目配置的 checkout URL 进行了真实打开验证：
- 年付 checkout URL 会进入 Creem checkout 页面
- 但页面报错：`Live payments are not enabled for your account`

这说明：
1. checkout 链接本身存在
2. 但 Creem 仍未为当前店铺开放 live 收款
3. 此时不应把官网购买按钮切回 live

## 项目侧已就绪项
### 1. license gateway 生产健康状态正常
- health endpoint 返回 200
- worker 可正常响应

### 2. license gateway 生产 secrets 已配置
已确认存在：
- `AGENTSHIELD_LICENSE_SIGNING_SEED`
- `CREEM_WEBHOOK_SECRET`
- `LICENSE_DELIVERY_FROM_EMAIL`
- `LICENSE_DELIVERY_REPLY_TO`
- `LICENSE_GATEWAY_ADMIN_PASSWORD`
- `RESEND_API_KEY`

### 3. 激活码链路代码已具备
- webhook 处理：`checkout.completed` / `subscription.paid` / `refund.created`
- 邮件发码：通过 Resend
- App 激活：本地校验签名并保存
- 后续在线校验：`/client/licenses/verify`

## 当前仍故意未开放的内容
官网 storefront 仍保持审核保护模式：
- 购买按钮 disabled
- 页面显示“审核中/待开放”提示

这是正确状态，因为此时若直接开放，用户会进入 Creem 的 `Payment Error` 页面。

## 你现在真正还差什么
只差外部平台状态：
- 等 `Review by Creem Team` 完成
- 等 `Payouts Enabled` 亮起

## 一旦通过后需要做什么
### Creem 侧
1. 再次打开任一 live checkout link
2. 确认不再出现 `Live payments are not enabled for your account`

### 项目侧
1. 将 storefront 中的购买按钮从 disabled 切换为 live checkout 链接
2. 重新部署 storefront
3. 跑一笔真实小额支付验收
4. 验证链路：
   - 支付成功
   - webhook 到达
   - 邮件收到激活码
   - App 粘贴激活成功
