# 45. Creem 后台零基础配置教程（正式商用）

- 日期：2026-03-16
- 适用版本：AgentShield v1.0.1 商用链路（Creem + License Gateway）

## 1. 你现在这页怎么填（创建产品）

你截图里的页面是 `Create Product -> Product Information`，先填这两个字段：

1. `Name`（名称）
   - 月付产品：`AgentShield Pro Monthly (30 Days)`
   - 年付产品：`AgentShield Pro Yearly (365 Days)`
   - 永久产品：`AgentShield Pro Lifetime`
2. `Description`（描述）
   - 月付：`Full Pro features for 30 days. Activation code delivered after payment.`
   - 年付：`Full Pro features for 365 days. Activation code delivered after payment.`
   - 永久：`Full Pro features with no expiry. Activation code delivered after payment.`

下一步（价格/计费）强烈建议：

1. `Billing Type` 选 `One-time`（一次性）
2. `Currency` 选你要卖的币种（例如 `USD`）
3. 分别建 3 个产品（Monthly / Yearly / Lifetime）

说明：当前 AgentShield 已兼容订阅事件，但你的“激活码售卖”模型用 one-time 最稳定、最直观。

## 2. 创建后要拿到的 5 个值

创建完 3 个产品后，你需要在 Creem 后台拿到以下值：

1. `Checkout URL`（月付）
2. `Checkout URL`（年付）
3. `Checkout URL`（永久）
4. `Webhook signing secret`
5. （可选）`Product ID`（做 product fallback 映射用）

## 3. Webhook 要怎么配

在 Creem Webhooks 里新增 endpoint：

1. URL 填：`https://你的公网网关域名/webhooks/creem`
2. 事件至少勾选：
   - `checkout.completed`
   - `subscription.paid`
   - `subscription.canceled`
   - `subscription.expired`
   - `refund.created`
3. 保存后复制 `Webhook signing secret`

## 4. 复制到本地环境文件（直接可用模板）

写入 `.env.public-sale.local`（或 CI 的同名变量）：

```env
VITE_CHECKOUT_MONTHLY_URL=https://checkout.creem.io/...
VITE_CHECKOUT_YEARLY_URL=https://checkout.creem.io/...
VITE_CHECKOUT_LIFETIME_URL=https://checkout.creem.io/...

AGENTSHIELD_LICENSE_GATEWAY_URL=https://你的公网网关域名
AGENTSHIELD_LICENSE_PUBLIC_KEY=你的base64url公钥

CREEM_WEBHOOK_SECRET=你的webhook secret
LICENSE_GATEWAY_ADMIN_PASSWORD=你的强密码
AGENTSHIELD_LICENSE_SIGNING_SEED=你的签名seed

CREEM_STRICT_BILLING_RESOLUTION=1
CREEM_SKU_BILLING_MAP_JSON={"AGSH_PRO_30D":"monthly","AGSH_PRO_365D":"yearly","AGSH_PRO_LIFETIME":"lifetime"}
# 可选（拿到 product id 再填）：
# CREEM_PRODUCT_BILLING_MAP_JSON={"prod_xxx":"monthly","prod_yyy":"yearly","prod_zzz":"lifetime"}

RESEND_API_KEY=re_xxx
LICENSE_DELIVERY_FROM_EMAIL="AgentShield <license@yourdomain.com>"
LICENSE_DELIVERY_REPLY_TO=support@yourdomain.com
```

## 5. 这一步完成后的验收标准

满足以下 5 条就算 Creem 侧配置完成：

1. 三个产品都已创建，且都是 `One-time`
2. 三条 `checkout URL` 已填入环境变量
3. webhook endpoint 指向 `/webhooks/creem`
4. webhook 事件至少包含上面 5 个事件
5. 本地执行 `pnpm run release:public:ready` 通过

## 6. 常见错误（你现在最容易踩的）

1. Checkout URL 还是 `example.com` 占位符
2. webhook 没选 `subscription.paid`（以后若改订阅会丢续期）
3. `CREEM_SKU_BILLING_MAP_JSON` 不是合法 JSON
4. 公网 URL 不是 `https://`
5. 网关 URL 用了临时隧道域名直接商用
