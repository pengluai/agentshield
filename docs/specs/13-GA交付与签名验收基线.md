# 13 - GA 交付与签名验收基线

更新日期: 2026-03-09  
状态: 可执行基线  
适用范围: AgentShield `macOS` / `Windows`

## 1. 执行摘要

本文件定义 AgentShield 从“试点部署”推进到“可公开交付”前必须完成的最后一公里验收。

当前仓库内已经可以真实完成的事项：

- MCP / Skill 安装管理
- 安全扫描
- 用户态运行时监控
- 自动 kill / quarantine / notify
- 受控启动本地 MCP
- 受控启动具备确定入口的本地 Skill
- 浏览器烟雾测试、前端集成测试、Rust 单元测试

当前仍依赖外部资产、不能在仓库内伪造完成的事项：

- Windows 代码签名证书
- Apple 代码签名与 notarization 账号/证书
- 干净 Win / mac 机器的实机回归

因此，本文件的目标不是虚报 GA，而是给出一套**可直接执行的 GA gate**，使后续交付只剩外部凭证和实机环境两类阻塞。

## 2. 对外表述边界

当前可对外使用的准确表述：

> AgentShield 现已具备 MCP / Skill 安装管理、安全扫描、运行时监控与自动化处置能力，适合试点部署。

当前不可对外使用的表述：

> 像系统杀毒软件一样全面拦截所有恶意行为。

补充说明：

- 如果当前阶段只是通过 GitHub 分发 `exe` / `dmg` 给试点用户，应走 `pilot` 分发链，不必把完整签名 / notarization 当成立刻阻塞项
- 如果要面向普通消费者公开售卖，应走 `public` 发布链，签名、公证、updater 必须完成

- Skill 的统一受控启动链已落地，但仅限可解析出本地入口的 Skill

## 4. 交付 Gate

### Gate A: Repo Local

通过条件：

- `pnpm run release:gate` 全部通过
- 没有新增 fake progress / random state / demo button
- 关键运行时链路在 UI 中可见且能触发真实命令

失败即禁止进入发布候选。

### Gate B: 实机回归

必须在干净环境分别执行：

- Windows 11
- macOS 14+

每台机器最少覆盖：

1. 首启向导与权限流程
2. 安装 OpenClaw
3. 安装商店 MCP
4. 发现手动新增 MCP
5. 发现手动新增 Skill
6. 受控启动 MCP
7. 受控启动 Skill
8. `blocked` 自动终止
9. `restricted + allowlist` 未授权外联自动终止并隔离
10. 卸载、升级、定位文件

### Gate C: 长稳与误杀

最低要求：

- `pnpm run test:soak` 至少跑 1 轮
- 真机持续运行 4 小时以上
- 收集误杀案例、漏报案例、崩溃案例
- 对每个案例给出：组件来源、trust_state、网络模式、处置结果、是否应调整策略

## 5. 风险登记

| 风险 | 概率 | 影响 | 分数 | 结论 |
| --- | --- | --- | --- | --- |
| 未签名直接发布导致安装信任差与更新链脆弱 | 4 | 5 | 20 | 发布前必须完成 |
| 缺少干净机回归导致平台兼容性问题 | 4 | 4 | 16 | 发布前必须完成 |
| Skill 目录格式不统一导致受控启动失败 | 3 | 4 | 12 | 已通过明确入口约定降低，但仍需在文档中解释 |
| restricted 未配置 allowlist 造成保护感知与真实能力不一致 | 3 | 3 | 9 | 已改为明确告警，继续观察 |
| 用户绕过 AgentShield 直接启动未知组件 | 3 | 4 | 12 | 通过发现后自动处置降低风险，但不属于内核级拦截 |

## 6. ADR

### ADR-013-1

决策：

- 不把“需要证书与真实机器”的发布 gate 伪装成已完成

备选方案：

- 继续只做源码与 UI 收口，并口头宣称 GA

结论：

- 拒绝。因为签名、notarization、实机回归都属于真实交付质量的一部分，不能用仓库内构建通过替代。

### ADR-013-2

决策：

- Skill 只支持可解析入口的受控启动

备选方案：

- 为所有 Skill 显示可启动并在运行时兜底失败

结论：

- 拒绝。运行时失败过晚，且会误导用户对产品能力的预期。
