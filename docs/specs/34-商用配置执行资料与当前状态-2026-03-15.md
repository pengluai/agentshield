## 4. 新增脚本说明
文件：

```bash
scripts/sync-github-public-sale-config.sh
```

作用：
- 读取 `.env.public-sale.local`
- 校验 checkout URL、license gateway URL、必填变量和密钥
- 在 `plan` 模式下输出将要同步哪些 GitHub Variables / Secrets
- 在 `--apply` 模式下直接同步到当前 GitHub 仓库

示例：

```bash
# 仅校验并输出计划
bash ./scripts/sync-github-public-sale-config.sh

# 指定 profile
bash ./scripts/sync-github-public-sale-config.sh --profile public

# 推送到 GitHub 仓库
bash ./scripts/sync-github-public-sale-config.sh --apply
```

验证结果：
- 使用当前 `.env.public-sale.local` 运行时，脚本会正确阻断 `example.com`
- 使用一份临时合规样例运行时，脚本能正确输出 GitHub 同步计划

## 5. 你还需要准备的真实值
下面这些值我不能替你凭空生成，因为它们来自你的外部平台账户：

- `VITE_CHECKOUT_MONTHLY_URL`

要求：

### 5.3 Cloudflare 登录态
二选一：
- 执行 `npx wrangler login`
- 或配置 `CLOUDFLARE_API_TOKEN`

## 6. 最短落地路径
### 6.1 如果你要继续用 GitHub 分发
1. 先在支付平台拿到 3 个真实 checkout URL
2. 先准备一个稳定公网 HTTPS 的 license gateway 地址
3. 把真实值填进 `.env.public-sale.local`
4. 运行：

```bash
bash ./scripts/sync-github-public-sale-config.sh --apply
pnpm run release:github:ready
```

第一成功检查点：
- 配置同步脚本通过
- `release:github:ready` 通过
- App 内“购买激活码”点击后跳到真实支付页

### 6.2 如果你要继续冲正式 public release
还需要补齐：
- Apple notarization / signing 相关值
- Windows 签名相关值

然后运行：

```bash
PUBLIC_RELEASE_PROFILE=public bash ./scripts/sync-github-public-sale-config.sh --profile public --apply
pnpm run release:public:ready
```

## 7. Troubleshooting
### 问题：脚本提示 `example.com`
原因：
- 你还没有把真实支付链接填进 `.env.public-sale.local`

### 问题：脚本提示 `trycloudflare.com`
原因：
- 你还在使用临时 tunnel 地址
- 正式商用必须换稳定 HTTPS 域名

### 问题：`wrangler whoami` 未登录
修复：

```bash
npx wrangler login
```

### 问题：GitHub 仓库配置还是旧值
修复：

```bash
bash ./scripts/sync-github-public-sale-config.sh --apply
```
