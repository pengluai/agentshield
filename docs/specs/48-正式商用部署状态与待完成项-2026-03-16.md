## 2. 当前还差哪一步才算“完整商用可卖”

### 2.1 还差 1 个外部配置：有效的邮件发送配置

当前真实测试结果显示：

- 付款链路是通的
- 激活码签发是通的
- 但是自动发邮件失败

失败原因已经明确验证过：

- 当前 `RESEND_API_KEY` 无效，Resend 返回 `401 invalid API key`

这意味着：

1. 你的用户现在付款后，后台可以生成激活码
2. 但是系统不能自动把激活码邮件发给买家
3. 所以现在还不适合直接公开卖给陌生用户

### 2.2 还需要你亲自准备的东西

你需要提供 2 个真实值：

1. 一个有效的 `Resend API Key`
2. 一个已经在 Resend 验证通过的发件地址或发件域

当前本地配置里的发件信息还是占位值：

- `AgentShield <license@example.com>`
- `support@example.com`

这不是正式商用可用的发件配置。

## 3. 你下一步怎么做

### 3.1 最短路径

1. 注册并登录 Resend
2. 在 Resend 里验证你的发件域名，或使用它允许的已验证发件地址
3. 创建一个新的 API Key
4. 把下面 3 个值给我，或者你自己填入本地环境文件：
   - `RESEND_API_KEY`
   - `LICENSE_DELIVERY_FROM_EMAIL`
   - `LICENSE_DELIVERY_REPLY_TO`
5. 我再帮你一键同步到 Cloudflare Worker，并复测真实邮件投递

### 3.2 如果你想自己填

需要改的文件：

1. `.env.license-gateway.local`
2. `.env.public-sale.local`

需要替换的键：

1. `RESEND_API_KEY=你的真实key`
2. `LICENSE_DELIVERY_FROM_EMAIL=你的真实已验证发件地址`
3. `LICENSE_DELIVERY_REPLY_TO=你的真实回复邮箱`

然后重新执行：

```bash
cd /Users/luheng/Downloads/ai01/agentshield
PUBLIC_RELEASE_PROFILE=pilot PUBLIC_REQUIRE_TAURI_RELEASE_CONFIG=0 bash ./scripts/public-sale-gate.sh
cd workers/license-gateway
npx wrangler secret bulk <(cat <<JSON
{
  "RESEND_API_KEY": "你的真实key",
  "LICENSE_DELIVERY_FROM_EMAIL": "你的真实发件地址",
  "LICENSE_DELIVERY_REPLY_TO": "你的真实回复邮箱"
}
JSON
)
npx wrangler deploy
```

说明：上面命令我已经验证过思路，实际值需要你自己的 Resend 账号提供。

## 4. 激活码现在怎么拿

### 4.1 自动方式

当 Creem 发来正式 `checkout.completed` webhook 时，Cloudflare Worker 会自动：

1. 创建订单记录
2. 生成激活码
3. 记录 license
4. 尝试发邮件

现在卡在第 4 步邮件发送。

### 4.2 手动方式

即使邮件还没修好，你仍然可以手动生成激活码：

```bash
cd /Users/luheng/Downloads/ai01/agentshield
./scripts/issue-public-license.sh --billing-cycle monthly --customer buyer@example.com
./scripts/issue-public-license.sh --billing-cycle yearly --customer buyer@example.com
./scripts/issue-public-license.sh --billing-cycle lifetime --customer buyer@example.com
```

用途：

1. 内测发码
2. 退款后人工补发
3. 博主合作时临时手工处理订单
