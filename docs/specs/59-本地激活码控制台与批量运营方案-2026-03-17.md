# 59-本地激活码控制台与批量运营方案-2026-03-17

- 日期：2026-03-17
- 适用范围：本地运行的 AgentShield 激活码签发、撤销、补发、实时追踪

## 1. 目标

给运营方一个本地控制台，支持：

1. 选择数量批量生成激活码
2. 自定义周期（monthly/yearly/lifetime）与有效天数
3. 单码撤销、重签、重发
4. 实时查看码状态和最近验证情况

## 2. 实现位置

1. 网关新增管理员签发接口：`POST /admin/licenses/issue`
2. 网关继续复用既有接口：
   - `GET /admin/licenses`
   - `POST /admin/licenses/:id/revoke`
   - `POST /admin/licenses/:id/reissue`
   - `POST /admin/orders/:id/resend`
3. 本地控制台：`scripts/license-ops-console.mjs`
4. 启动命令：`pnpm run license:ops:console`

## 3. 运行步骤

1. 启动网关（本地或远程）  
   `pnpm run gateway:dev`
2. 启动控制台  
   `pnpm run license:ops:console`
3. 打开浏览器  
   `http://127.0.0.1:8790`

## 4. 运营场景

### 4.1 送博主几十个码

1. 在控制台选择数量（例如 50）
2. 选择周期与天数
3. 输入单邮箱或多邮箱列表
4. 点击生成，导出 CSV 发给运营同事

### 4.2 撤回/补发

1. 在实时追踪表中按邮箱过滤
2. 对指定 license 执行：
   - `撤销`
   - `重签`（返回新码）
   - `重发`（按 order 重发）

### 4.3 中国用户购买

1. 用户通过 Creem Checkout 下单
2. `checkout.completed` webhook 到网关自动发码
3. 如用户没收到邮件，可在控制台用 `重发`

## 5. 追踪字段

本次补充字段：

1. `verify_count`
2. `last_verified_at`
3. `last_verified_device`

这些字段在客户端在线验码时实时更新，用于判断码是否活跃、是否异常共享。

## 6. 验收检查

1. 能批量生成并拿到 `AGSH.*` 激活码
2. `GET /admin/licenses` 能看到新签发记录
3. 撤销后客户端在线验码状态变更为 `revoked`
4. 控制台自动刷新可看到 `verify_count` 增长
