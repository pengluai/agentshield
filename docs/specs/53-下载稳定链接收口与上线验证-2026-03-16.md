## 2. 关键官方结论

### 2.1 GitHub latest release 限制

GitHub 官方文档与官方 API 语义决定：

1. `latest release` 不包含 draft / prerelease。
2. 因此，如果当前下载仓库发的是 prerelease，不能依赖 `releases/latest/download/<asset>` 作为稳定下载主方案。

所以本次落地采用：

- `固定 tag + 稳定资产名`
- 官网按钮只打到 Cloudflare storefront 稳定路由

## 2.2 Cloudflare 推荐实现

Cloudflare 官方支持在 Worker 中先处理自定义下载路由，再回落 `env.ASSETS.fetch(request)` 提供静态页面。

本次 storefront 继续沿用该模式。
