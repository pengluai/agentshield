# 00 - 总架构与协议文档

更新日期: 2026-03-12  
适用范围: AgentShield GitHub 直装版（macOS / Windows）

## 1. 产品边界

1. 只围绕可调用 MCP/Skill 的 AI 工具做发现、扫描、受控管理、运行时守卫。
2. 不做系统内核级全局拦截，不承诺“替代系统杀毒软件”。
3. 对外默认文案使用后果导向，不以专业术语作为首层信息。

## 2. 逻辑架构

1. 前端：React + Zustand，负责资产展示、审批交互、手动/一键流程入口。
2. 后端：Tauri + Rust commands，负责真实命令执行、策略判定、审批验票、落盘。
3. 数据：本地 `~/.agentshield`（事件、组件、许可证、规则状态）。

## 3. 核心协议与契约

1. 所有高风险管理动作必须走“请求审批 -> 获得一次性票据 -> 后端验票执行”。
2. 免费/付费边界以“后端门禁”生效，不能只依赖前端 UI。
3. 规则更新频率分层：免费低频手动，试用/付费高频自动。

## 4. 关键命令入口

- 扫描与修复：`scan.rs`
- 商店与安装管理：`store.rs`
- OpenClaw 管理：`install.rs`、`ai_orchestrator.rs`
- 运行时守卫：`runtime_guard.rs`
- 许可证：`license.rs`
- 规则更新与通知：`notification.rs`
- 密钥保险库：`vault.rs`

## 5. 验收门槛

1. Repo-local 质量门必须持续通过：`typecheck/lint/test/cargo test/playwright/release:gate`。
2. 高危动作必须后端验票，任何 UI 绕过必须失败。
3. 商用前必须完成干净机真机回归（见 `14-干净机实机回归清单.md`）。
