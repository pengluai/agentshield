# 46. Creem 商用修复完成与测试验收

- 日期：2026-03-16
- 目标：将 Creem 支付与许可证链路修复到可正式商用，并完成全量代码审查与功能测试

## 1. 本轮修复项（已完成）

1. Webhook 事件覆盖增强（商用关键）
   - 新增支持：`subscription.paid`、`subscription.canceled`、`subscription.expired`
   - 保留：`checkout.completed`、`refund.created`
   - 文件：`scripts/license-gateway.mjs`

2. Webhook 幂等增强（防重放/防重复延长）
   - 增加 `event_id` 去重记录：`processed_webhook_events`
   - 同一事件 ID 重试时直接 200 + duplicate，不重复改状态
   - 文件：`scripts/license-gateway.mjs`

3. 订阅续期逻辑落地
   - `subscription.paid` 会延长许可证有效期（按 billing cycle）
   - 续期锚点采用 `max(now, current_expiry)`，避免丢天数
   - 文件：`scripts/license-gateway.mjs`

4. 计费周期解析从“字符串猜测”升级为“强映射优先”
   - 新增环境变量：
     - `CREEM_SKU_BILLING_MAP_JSON`（必填）
     - `CREEM_PRODUCT_BILLING_MAP_JSON`（可选）
     - `CREEM_STRICT_BILLING_RESOLUTION`（默认 1）
   - strict 模式下无法解析会直接报错，不再静默误判
   - 文件：`scripts/license-gateway.mjs`

5. 发布门禁与配置同步脚本升级
   - `public-sale-gate` 新增 JSON 映射变量校验
   - `sync-github-public-sale-config` 新增映射变量校验和同步
   - 文件：
     - `scripts/public-sale-gate.sh`
     - `scripts/sync-github-public-sale-config.sh`

6. CI 工作流补齐新变量注入
   - `publish-pilot-artifacts.yml`
   - `publish-signed-release.yml`

7. 客户端在线校验兼容修复（续期生效）
   - 当网关返回 active 且远端有效期更新后，可恢复本地 active 状态
   - 避免“本地 expired 后即使远端续期仍保持 expired”的问题
   - 文件：`src-tauri/src/commands/license.rs`

8. 文档与示例环境文件更新
   - `.env.license-gateway.example`
   - `.env.public-sale.example`
   - `.env.public-release.example`
   - `README.md`
   - 新增教程：`45-Creem后台零基础配置教程-2026-03-16.md`

9. 新增自动化测试（高风险路径）
   - 测试文件：`src/__tests__/license-gateway-webhook.test.ts`
   - 覆盖：
     - `subscription.paid` 幂等（同 event id 只处理一次）
     - 无 active license 时不误发权益

## 4. 仍需你手动完成的外部配置（非代码）

1. 在 Creem 后台创建 3 个产品并拿到 3 条 checkout URL
2. 配置 webhook endpoint 为：
   - `https://你的网关域名/webhooks/creem`
3. 在 Creem 勾选事件：
   - `checkout.completed`
   - `subscription.paid`
   - `subscription.canceled`
   - `subscription.expired`
   - `refund.created`
4. 把真实值写入 `.env.public-sale.local` / GitHub Variables & Secrets
