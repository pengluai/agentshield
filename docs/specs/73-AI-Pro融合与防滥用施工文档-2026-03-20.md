## 2. Proxy 配额管控（Cloudflare KV）

> 基于 Cloudflare Workers KV 官方文档:
> - KV put/get: developers.cloudflare.com/kv
> - TTL 自动过期: KV put 支持 `expirationTtl` 参数

### 2.1 wrangler.jsonc 新增 KV binding

```jsonc
{
  // ... 现有配置 ...
  "kv_namespaces": [
    {
      "binding": "USAGE_KV",
      "id": "<创建后填入>"
    }
  ]
}
```

创建 KV namespace:
```bash
npx wrangler kv namespace create "USAGE_KV"
# 返回 id，填入 wrangler.jsonc
```

### 2.2 配额方案

| 用户类型 | 每日上限 | 每月上限 | 每分钟 Rate Limit |
|---------|---------|---------|-----------------|
| 永久免费版 | **0**（完全拦截） | 0 | N/A |
| Pro 试用（14天） | 50 | 500 | 10 |
| Pro 月付 | 100 | 2000 | 10 |
| Pro 年付/终身 | 200 | 5000 | 10 |

### 2.3 实现逻辑（Proxy index.mjs 中）

```javascript
// KV key 格式:
// 每日: "usage:daily:{license_id}:{YYYY-MM-DD}"  TTL: 48h
// 每月: "usage:monthly:{license_id}:{YYYY-MM}"   TTL: 35天

async function checkAndIncrementQuota(env, licenseId, plan) {
  const now = new Date();
  const dayKey = `usage:daily:${licenseId}:${now.toISOString().slice(0, 10)}`;
  const monthKey = `usage:monthly:${licenseId}:${now.toISOString().slice(0, 7)}`;

  // 获取当前用量
  const dailyCount = parseInt(await env.USAGE_KV.get(dayKey) || '0');
  const monthlyCount = parseInt(await env.USAGE_KV.get(monthKey) || '0');

  // 配额表
  const limits = {
    trial: { daily: 50, monthly: 500 },
    monthly: { daily: 100, monthly: 2000 },
    yearly: { daily: 200, monthly: 5000 },
    lifetime: { daily: 200, monthly: 5000 },
  };
  const limit = limits[plan] || limits.monthly;

  if (dailyCount >= limit.daily) {
    return { allowed: false, reason: `Daily limit reached (${limit.daily}/day)` };
  }
  if (monthlyCount >= limit.monthly) {
    return { allowed: false, reason: `Monthly limit reached (${limit.monthly}/month)` };
  }

  // 递增（KV 不支持原子递增，用 put 覆盖，可能有微小并发误差，可接受）
  await env.USAGE_KV.put(dayKey, String(dailyCount + 1), { expirationTtl: 172800 }); // 48h
  await env.USAGE_KV.put(monthKey, String(monthlyCount + 1), { expirationTtl: 3024000 }); // 35天

  return {
    allowed: true,
    daily_used: dailyCount + 1,
    daily_limit: limit.daily,
    monthly_used: monthlyCount + 1,
    monthly_limit: limit.monthly,
  };
}
```

## 3. HMAC 请求签名（反代防护）

> 基于 Cloudflare Workers 官方示例:
> developers.cloudflare.com/workers/examples/signing-requests

### 3.1 原理

```
App 端（Rust）:
  signature = HMAC-SHA256(signing_key, license_id + timestamp + nonce)
  Header: X-Signature: {timestamp}-{nonce}-{base64(signature)}

Proxy 端（Worker）:
  提取 timestamp, nonce, signature
  验证 signature = HMAC-SHA256(signing_key, license_id + timestamp + nonce)
  检查 timestamp 未过期（±300秒）
  检查 nonce 未重复使用（KV 存 nonce，TTL 5分钟）
```

### 3.2 签名密钥

```
App 端: 编译进 Rust 二进制的常量（反编译成本 > 滥用收益）
Proxy 端: wrangler secret put SIGNING_SECRET
两边用同一个密钥
```

### 3.3 App 端签名（Rust）

```rust
use hmac::{Hmac, Mac};
use sha2::Sha256;
use base64::{Engine, engine::general_purpose::URL_SAFE_NO_PAD};
use rand::Rng;

const SIGNING_SECRET: &[u8] = b"agentshield-proxy-signing-key-v1-CHANGE-IN-PRODUCTION";

fn sign_request(license_id: &str) -> String {
    let timestamp = chrono::Utc::now().timestamp();
    let nonce: u64 = rand::thread_rng().gen();

    let data = format!("{license_id}{timestamp}{nonce}");
    let mut mac = Hmac::<Sha256>::new_from_slice(SIGNING_SECRET).expect("HMAC key");
    mac.update(data.as_bytes());
    let result = mac.finalize().into_bytes();
    let sig = URL_SAFE_NO_PAD.encode(result);

    format!("{timestamp}-{nonce}-{sig}")
}
```

### 3.4 Proxy 端验证（Worker）

```javascript
import { Buffer } from "node:buffer";

const encoder = new TextEncoder();

async function verifySignature(env, licenseId, signatureHeader) {
  if (!signatureHeader) return false;

  const parts = signatureHeader.split('-');
  if (parts.length < 3) return false;

  const timestamp = parseInt(parts[0]);
  const nonce = parts[1];
  const receivedSig = parts.slice(2).join('-'); // base64 may contain -

  // Check timestamp freshness (±300 seconds)
  const now = Math.floor(Date.now() / 1000);
  if (Math.abs(now - timestamp) > 300) return false;

  // Check nonce not replayed (KV with 5min TTL)
  const nonceKey = `nonce:${nonce}`;
  if (await env.USAGE_KV.get(nonceKey)) return false;
  await env.USAGE_KV.put(nonceKey, '1', { expirationTtl: 300 });

  // Verify HMAC
  const secretKeyData = encoder.encode(env.SIGNING_SECRET);
  const key = await crypto.subtle.importKey(
    "raw", secretKeyData, { name: "HMAC", hash: "SHA-256" }, false, ["verify"]
  );

  const dataToAuth = `${licenseId}${timestamp}${nonce}`;
  const receivedMac = Buffer.from(receivedSig, 'base64url');

  return await crypto.subtle.verify(
    "HMAC", key, receivedMac, encoder.encode(dataToAuth)
  );
}
```

## 4. Semantic Guard 迁移到 Proxy

### 4.1 现状

`semantic_guard.rs` 直接调 DeepSeek API:
- 常量: `DEEP_REVIEW_BASE_URL = "https://api.deepseek.com"`
- 常量: `DEEP_REVIEW_MODEL = "deepseek-chat"`
- 用户需要在系统钥匙串中存 DeepSeek API Key
- Pro 功能 gate: `checkFeature('semantic_guard')`

### 4.2 迁移方案

**选项 A（推荐）：Proxy 新增 `/v1/semantic-review` 端点**

Proxy 端新增一个专用路由，接收扫描结果，用内置 Key 调 MiniMax 做语义分析，返回结构化结果。

**选项 B：复用 `/v1/chat/completions`**

semantic_guard.rs 改为通过 `pro_ai_chat` 调 Proxy，自己构造 prompt 发送扫描结果给 AI 分析。

**推荐选项 B** — 改动最小，不需要 Proxy 新增端点。

### 4.3 改动清单

**semantic_guard.rs:**
1. 现有逻辑：直接 `reqwest::post(DEEP_REVIEW_BASE_URL)` → 改为调 `pro_ai_chat` 的 proxy 路径
2. 或者更简单：在 `run_semantic_review` 中，检查是否有 Proxy URL 配置，如果有就走 Proxy，否则用自配 Key（向后兼容）

**settings-page.tsx:**
1. Pro 用户的 AI 配置区域显示："✅ Pro 内置 AI 已启用（MiniMax M2.7），无需配置"
2. 只有"自定义"模式才显示 Key/Model 输入框

## 5. 完整 Proxy 逻辑流程（更新后）

```
请求到达 Proxy
  ↓
1. 检查 X-License-ID 存在 → 无 → 401
  ↓
2. 验证 X-Signature (HMAC) → 失败 → 403 "Invalid signature"
  ↓
3. 验证 License 有效性 → 无效 → 403
  ↓
4. 检查 plan → free → 403 "Upgrade to Pro"
  ↓
5. Rate Limit (10/min) → 超限 → 429
  ↓
6. 配额检查 (daily/monthly KV) → 超限 → 429 + 剩余配额信息
  ↓
7. 转发到 MiniMax → 返回结果
  ↓
8. 响应头附带配额信息:
   X-Quota-Daily-Used: 12
   X-Quota-Daily-Limit: 100
   X-Quota-Monthly-Used: 156
   X-Quota-Monthly-Limit: 2000
```
