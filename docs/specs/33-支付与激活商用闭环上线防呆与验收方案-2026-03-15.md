# 33-支付与激活商用闭环上线防呆与验收方案-2026-03-15

## 1. Executive Summary
- 目标：将 AgentShield 的“试用 + 支付 + 发码 + 激活 + 退款撤销”闭环提升到可稳定商用的上线基线。
- 当前核心缺口（基于代码与 CI 实际结果）：
  - `VITE_CHECKOUT_*` 可被占位地址（如 `example.com`）通过 gate，导致发布包内购买按钮不可收款。
  - `AGENTSHIELD_LICENSE_GATEWAY_URL` 可被临时 tunnel 地址（如 `trycloudflare.com`）通过 gate，导致在线验码与 webhook 稳定性不足。
- 本次改造范围：只做上线防呆与质量门禁增强，不重写支付架构，不变更既有 license 签名协议。

## 2. Problem Statement and Scope
- 问题陈述：
  - `scripts/public-sale-gate.sh` 目前只校验 HTTPS 前缀，未阻断占位域名或临时隧道域名。
  - `UpgradePro` 页面在 URL 非空时直接打开外链，无法识别明显占位链接。
- 范围内：
  - 发布门禁脚本增强（pilot/public 差异化策略）。
  - 前端购买入口防呆校验（减少用户点了“购买”却打不开真实收款页）。
  - 新增或更新自动化测试与手工验收清单。
- 范围外：
  - 第三方支付账户开通、税务配置、法务材料提交。
  - 许可证网关迁移到新云服务商。

## 3. Current State and Constraints
- 已有能力（代码已实现）：
  - 离线激活码本地验签：`src-tauri/src/commands/license.rs`
  - 在线校验端点：`POST /client/licenses/verify`（由 `scripts/license-gateway.mjs` 提供）
  - Lemon Squeezy webhook 验签与退款撤销逻辑：`scripts/license-gateway.mjs`
- 约束：
  - 不破坏现有 `release:github:ready` 与 `release:public:ready` 流程。
  - 必须保持对零基础发布者可执行：错误信息明确，失败即阻断（fail fast）。

## 4. Target Architecture Overview
- 新增“配置真实性门禁层”：
  - 在发布前 gate 阶段识别并阻断占位 checkout URL。
  - 在 public profile 下阻断临时 tunnel 网关地址（`trycloudflare.com` 等）。
- 前端“购买入口前置校验层”：
  - 若发现占位 URL，直接提示“购买链接未配置/无效”，不跳转。
- 运行时链路不变：
  - 支付成功 -> webhook -> 发码 -> 用户粘贴激活码 -> 在线状态周期校验。

## 5. Detailed Component Design
### 5.1 发布门禁（`scripts/public-sale-gate.sh`）
- 新增 URL 占位检测：
  - 拒绝 `example.com`、`localhost`、`127.0.0.1`、`invalid` 等明显非生产域名。
- 新增网关域名策略：
  - `PUBLIC_RELEASE_PROFILE=public` 时，拒绝 `trycloudflare.com` 等临时域名。
  - `pilot` 保持允许（用于测试分发）。
- 输出清晰错误：
  - 明确提示哪个变量不合规、为何不合规、如何修复。

### 5.2 购买入口防呆（`src/components/pages/upgrade-pro.tsx`）
- 在 `handlePurchase` 中新增 `isCheckoutUrlUsable` 判断：
  - URL 空值或明显占位域名 -> 提示错误并中断跳转。
- 保持现有 `custom_data` 附带逻辑不变。

### 5.3 测试与回归
- 脚本层：
  - 新增 shell 测试脚本（或最小可执行校验脚本）覆盖占位 URL 与临时网关域名场景。
- 前端层：
  - 新增 `upgrade-pro` 单测：占位 URL 下点击购买应显示错误，不应调用 `openExternalUrl`。

## 6. Data Model and Interface Contracts
- 环境变量契约（保持不变，增加语义约束）：
  - `VITE_CHECKOUT_MONTHLY_URL`, `VITE_CHECKOUT_YEARLY_URL`, `VITE_CHECKOUT_LIFETIME_URL`
  - `AGENTSHIELD_LICENSE_GATEWAY_URL`
- 新语义约束：
  - URL 不得为占位域名。
  - `public` 发布 profile 下 `AGENTSHIELD_LICENSE_GATEWAY_URL` 不得使用临时隧道域名。

## 7. Non-Functional Requirements
- Security：
  - 上线包不得包含明显伪配置支付链接。
  - webhook 验签实现维持原始请求体 + HMAC + 恒定时序比较。
- Reliability：
  - Gate 在配置不合规时必须失败退出（非 warning）。
- Operability：
  - 错误信息可直接指导配置修复，减少人工排障成本。

## 8. ADRs
### ADR-33-01：先加“上线防呆”再扩展功能
- 决策：优先增强 gate 与入口防呆，不在本次引入新支付通道或复杂 seat 管理。
- 备选方案：
  - 直接推进更多商业功能（风险：现有发布质量未封口，可能放大商用事故）。
- 结论：低风险、快收口、可立即提升商用稳定性。

### ADR-33-02：public 与 pilot 分级校验
- 决策：`public` 严格阻断临时网关；`pilot` 允许测试配置。
- 原因：兼顾测试效率与正式商用可靠性。

## 9. Risk Register
- 风险 1：规则过严导致测试发布受阻。`P=2, I=3, Score=6`
  - 缓解：仅在 `public` 启用最严格网关域名限制。
- 风险 2：漏判占位域名。`P=3, I=4, Score=12`
  - 缓解：规则显式列出常见占位域名并新增回归测试。
- 风险 3：前端判断与 gate 判断不一致。`P=2, I=4, Score=8`
  - 缓解：抽出一致规则或同步用例覆盖。

## 11. Runbook and Observability Baseline
- 发布前必跑：
  - `pnpm run release:github:ready`（pilot）
  - `pnpm run release:public:ready`（public）
- 回滚策略：
  - 若新规则误伤，先通过 `pilot` 验证，定位后仅回退新增校验片段，不回滚既有安全逻辑。

## 12. Acceptance Criteria
- AC-1：占位 checkout URL 在 gate 中被明确阻断。
- AC-2：`public` profile 下，临时 tunnel 网关地址被阻断。
- AC-3：前端购买按钮遇到占位 URL 时给出错误，不触发外部跳转。
- AC-4：`lint + typecheck + vitest + e2e + cargo test + release gate` 全通过。
- AC-5：发布产物验证时，使用真实支付链接可完成“购买->发码->激活”人工冒烟。
