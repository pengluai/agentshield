# 10 - AI 智能配置系统 (AI-Powered Smart Setup)

> Pro 功能 · 版本 1.1.0

## 1. 概述

AI 智能配置系统是 AgentShield Pro 的特色功能，通过**混合架构**（确定性 Rust 命令 + LLM 错误诊断）实现一键安装配置 OpenClaw，用户只需点击"下一步"即可完成全部安装、配置、安全加固流程。

### 设计原则
- **确定性优先**: 95% 的操作由 Rust 确定性代码执行（可靠、快速、零 LLM 成本）
- **AI 仅处理异常**: LLM 仅在安装出错时介入，提供诊断和修复建议
- **零认知负担**: 用户无需理解技术细节，只需选择和填入必要信息
- **渐进式增强**: Free 用户看到手动说明，Pro 用户获得 AI 自动诊断

## 2. 架构

```
┌─────────────────────────────────────────────────┐
│                  OpenClaw Wizard UI              │
│  Step 0 → Step 1 → Step 2 → Step 3 → Step 4    │
│  检测      环境     安装     加固     渠道连接   │
└───────────────────────┬─────────────────────────┘
                        │ invoke()
┌───────────────────────┴─────────────────────────┐
│              ai_orchestrator.rs                   │
│                                                   │
│  execute_install_step()    ← 确定性 Rust 命令     │
│    ├── check_node          (which::which)         │
│    ├── install_openclaw    (npm install -g)       │
│    ├── run_onboard         (openclaw onboard)     │
│    ├── configure_channel   (写 JSON 配置)         │
│    ├── setup_mcp           (注入 MCP 配置)        │
│    ├── harden_permissions  (chmod 600)            │
│    └── verify_install      (openclaw --version)   │
│                                                   │
│  ai_diagnose_error()       ← 仅出错时调用 LLM    │
│  test_ai_connection()      ← 设置页面测试连接     │
└───────────────────────┬─────────────────────────┘
                        │ reqwest (仅错误诊断)
                        ▼
              ┌─────────────────┐
              │  LLM API        │
              │  DeepSeek/Gemini│
              └─────────────────┘
```

## 3. LLM 推荐方案

| 服务商 | 模型 | Input 价格 | Output 价格 | 推荐理由 |
|--------|------|-----------|------------|---------|
| **DeepSeek** (推荐) | deepseek-chat (V3.2) | $0.28/MTok | $0.42/MTok | 最佳性价比，缓存命中仅 $0.028 |
| Gemini | gemini-2.0-flash | $0.10/MTok | $0.40/MTok | 有免费额度，适合测试 |
| OpenAI | gpt-4o-mini | $0.15/MTok | $0.60/MTok | 通用兼容 |

**成本估算**: 典型安装流程中，LLM 仅在出错时调用（~500 输入 + ~300 输出 token），每次诊断成本约 $0.0003。成功安装使用零 LLM token。

## 4. Tauri 命令接口

### execute_install_step
```rust
#[tauri::command]
async fn execute_install_step(
    step_id: String,
    channel_id: Option<String>,
    token: Option<String>,
    platform_ids: Option<Vec<String>>,
) -> Result<StepResult, String>
```

StepResult 包含 `needs_ai_help: bool` 标志，前端据此决定是否触发 AI 诊断。

### ai_diagnose_error
```rust
#[tauri::command]
async fn ai_diagnose_error(
    provider: String, api_key: String,
    model: Option<String>, base_url: Option<String>,
    error_context: String, step_name: String,
) -> Result<AiDiagnosis, String>
```

### test_ai_connection
```rust
#[tauri::command]
async fn test_ai_connection(
    provider: String, api_key: String,
    model: Option<String>, base_url: Option<String>,
) -> Result<AiConnectionResult, String>
```

## 5. 设置页面 - AI 模型配置

位于 设置 → AI 智能配置，包含：
- 服务商选择（DeepSeek / Gemini / OpenAI / 自定义）
- API 密钥输入（带服务商特定提示）
- 模型选择（带默认推荐）
- 自定义端点（仅自定义服务商显示）
- 测试连接按钮

配置存储在 settingsStore (Zustand)，字段：
- `aiProvider`, `aiApiKey`, `aiModel`, `aiBaseUrl`, `aiConnectionTested`

## 6. 安全考虑

- API 密钥存储在本地 Zustand store，不上传到任何远程服务器
- 发送给 LLM 的错误信息经过脱敏处理（不包含完整文件路径中的用户名）
- channel token 配置文件权限设为 600（仅用户可读写）
- MCP 配置注入使用 JSON 合并策略，不覆盖已有配置
