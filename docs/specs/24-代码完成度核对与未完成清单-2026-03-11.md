## 1. 执行摘要

按当前仓库代码与实测结果，AgentShield 已具备“可真实使用”的主干能力：

1. 发现支持 `MCP / Skill` 的 AI 工具并建立资产视图。
2. 扫描风险、逐项修复、付费一键修复（含后端验票）。
3. OpenClaw 一键安装/更新/卸载（真实命令执行）。
4. 商店按指定宿主写入配置、更新、卸载（真实配置落盘）。
5. 运行时守卫对“受控启动 + 新外联”已形成实时审批链路。

但你要求的“零基础极简商用态”仍有关键缺口，尤其是：

1. 默认文案仍大量暴露 `MCP / Skill` 术语，未完全后果导向。
2. 对第三方宿主外部动作（删邮件/提交支付/网页提交/批量改写）仍需继续扩大生态覆盖验证。
3. 商店托管覆盖率仍有“手动来源跳转”条目，未达“全托管一键”。
4. 生产级 License Gateway 运维闭环（对账/吊销/回滚）仍需强化。

## 7. 验收标准（用于下一轮开发）

1. 用户从首页开始，3 步内完成扫描并看到“每个 AI 工具 -> 每个组件 -> 风险后果 -> 可执行动作”。
2. 免费用户可以逐项修复，付费用户可以一键修复，且每一步都是真实执行结果。
3. 运行时高危行为在触发时右下角弹出审批卡片，不打断主流程；拒绝后必须确实阻断。
4. 任一风险文案默认不出现专业术语；点击“查看详情”才展示技术字段。
5. “打开系统设置”失败时可给出可执行手动路径并带复检按钮，复检结果与系统实际一致。
