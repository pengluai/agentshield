## 2. 你现在怎么拿激活码

在本机执行：

```bash
cd /Users/luheng/Downloads/ai01/agentshield
pnpm run license:issue:public -- --billing-cycle monthly --customer buyer@example.com --days 31
```

其他常用命令：

```bash
pnpm run license:issue:public -- --billing-cycle yearly --customer buyer@example.com --days 366
pnpm run license:issue:public -- --billing-cycle lifetime --customer buyer@example.com
pnpm run license:public-key
```

说明：

1. 生成结果里会返回一条以 `AGSH.` 开头的真实激活码。
2. 这条激活码可直接给客户在 App 内粘贴激活。
3. 当前本地签名 seed 与 public key 已匹配，已实测可签发有效码。

## 4. 自动支付发码为什么还没彻底放开

当前还缺真实外部配置：

1. `VITE_CHECKOUT_MONTHLY_URL`
2. `VITE_CHECKOUT_YEARLY_URL`
3. `VITE_CHECKOUT_LIFETIME_URL`
4. 稳定公网 HTTPS `AGENTSHIELD_LICENSE_GATEWAY_URL`

因为源码仓库里的发布 gate 已经做了防呆，所以：

1. 占位 checkout URL 会被阻断
2. 临时 tunnel gateway 会被阻断

这意味着当前代码已经能防止“误把假支付配置发上线”。
