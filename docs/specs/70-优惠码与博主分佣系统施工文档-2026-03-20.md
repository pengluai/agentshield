## 2. Creem API 官方参考（已验证）

### 2.1 创建折扣码

> 来源: Creem 官方文档 `POST /v1/discounts` — Context7 verified

```bash
curl -X POST https://api.creem.io/v1/discounts \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "博主A-30off",
    "code": "BLOGGER_ABC",
    "type": "percentage",
    "percentage": 30,
    "duration": "once",
    "applies_to_products": [
      "prod_2T8qrIwLHQ3AlG4KtTB849",
      "prod_7kbjugsRm1gGN6lKXOR1NG",
      "prod_4rh2nT74Cqk4IQ5EfvcjbH"
    ],
    "expiry_date": "2027-12-31T23:59:59Z",
    "max_redemptions": 1000
  }'
```

**参数说明（官方）:**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 内部名称 |
| `code` | string | 是 | 用户输入的折扣码 |
| `type` | string | 是 | `percentage` 或 `fixed` |
| `percentage` | number | 条件 | 百分比折扣值（如 30 表示 30%） |
| `amount` | number | 条件 | 固定金额折扣（单位: 分） |
| `currency` | string | 条件 | 固定折扣的货币 |
| `duration` | string | 是 | `once` / `forever` / `repeating` |
| `durationInMonths` | number | 条件 | `repeating` 时必填 |
| `applies_to_products` | string[] | 否 | 产品 ID 列表，空则全部适用 |
| `expiry_date` | string | 否 | ISO 8601 过期时间 |
| `max_redemptions` | number | 否 | 最大使用次数 |

**响应:**
```json
{
  "id": "disc_abc123xyz",
  "name": "博主A-30off",
  "code": "BLOGGER_ABC",
  "type": "percentage",
  "percentage": 30,
  "duration": "once",
  "applies_to_products": ["prod_xxx"],
  "expiry_date": "2027-12-31T23:59:59Z",
  "max_redemptions": 1000,
  "created_at": "2026-03-20T10:00:00Z"
}
```

### 4.4 新增 API 端点

#### 4.4.1 公开接口：验证优惠码

```javascript
// POST /api/promos/validate  (不需要 admin 认证)
// Body: { "code": "BLOGGER_ABC" }
// Response: { valid, discount_pct, message }

if (req.method === 'POST' && pathname === '/api/promos/validate') {
  const body = JSON.parse((await readRawBody(req)).toString('utf8'));
  const code = String(body?.code ?? '').trim().toUpperCase();
  const promo = state.promo_codes.find(
    (p) => p.code === code && p.active
  );

  if (!promo) {
    sendJson(res, 200, { valid: false, message: '无效的优惠码' });
    return;
  }
  if (promo.expires_at && new Date(promo.expires_at) < new Date()) {
    sendJson(res, 200, { valid: false, message: '优惠码已过期' });
    return;
  }
  if (promo.max_uses && promo.times_used >= promo.max_uses) {
    sendJson(res, 200, { valid: false, message: '优惠码已用完' });
    return;
  }

  const affiliate = state.affiliates.find((a) => a.id === promo.affiliate_id);

  sendJson(res, 200, {
    valid: true,
    discount_pct: promo.discount_pct,
    affiliate_id: promo.affiliate_id,
    affiliate_name: affiliate?.name ?? null,
    message: `优惠 ${promo.discount_pct}% 折扣`,
  });
  return;
}
```

### 5.1 新增优惠码输入框

在购买按钮上方增加优惠码输入区：

```tsx
// 新增 state
const [promoCode, setPromoCode] = useState('');
const [promoResult, setPromoResult] = useState<{
  valid: boolean;
  discount_pct: number;
  affiliate_id: string;
  message: string;
} | null>(null);
const [validatingPromo, setValidatingPromo] = useState(false);

// 验证函数
const validatePromo = async () => {
  if (!promoCode.trim()) return;
  setValidatingPromo(true);
  try {
    const resp = await fetch(`${LICENSE_GATEWAY_URL}/api/promos/validate`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ code: promoCode.trim() }),
    });
    const data = await resp.json();
    setPromoResult(data);
  } catch {
    setPromoResult({ valid: false, discount_pct: 0, affiliate_id: '', message: '网络错误' });
  } finally {
    setValidatingPromo(false);
  }
};
```

## 6. 防滥用措施

| 威胁 | 措施 | 实现层 |

| 优惠码超限使用 | `max_redemptions` 参数 | Creem 原生 |

## 9. 关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 折扣实现 | Creem 原生 Discount API | 官方支持，无需自建计算逻辑 |
| 分佣追踪 | 自建在 license-gateway | 已有 webhook 处理链，最小改动 |
| 数据存储 | JSON 文件扩展 | 与现有架构一致，初期规模足够 |
| 第三方分销平台 | 不使用 | 省月费($49+/月)，自建够用 |
| 打款 | 手动（CSV导出后打款） | 初期量不大，自动化 ROI 不高 |
| 锁定期 | 30天 | 行业标准，防退款骗佣 |
| 多级分佣 | 不做 | 需求明确：不做 |

## 11. 快速操作指南（给你自己看）

### 创建一个新博主的完整步骤

```
1. 调用 POST /admin/affiliates
   Body: {
     "name": "某B站博主",
     "email": "blogger@xxx.com",
     "platform": "bilibili",
     "commission_pct": 20,
     "discount_pct": 30,
     "promo_code": "BSTATION_XXX",
     "payout_method": { "type": "wechat", "account": "wxid_xxx" }
   }

2. 后台自动:
   → 调用 Creem API 创建折扣码 BSTATION_XXX (30% off)
   → 创建 affiliate 记录
   → 创建 promo_code 记录

3. 把优惠码 BSTATION_XXX 给到博主

4. 博主发视频: "用我的优惠码 BSTATION_XXX 下单打7折！"

5. 每月查看 GET /admin/commissions → 导出 CSV → 给博主打款
```

### 查看某博主分佣

```
GET /admin/affiliates
→ 响应包含每个博主的:
  - 总转化次数
  - 总销售额
  - 总分佣（含已打款+待打款）
  - 待打款金额
```

## 12. 风险与备选

| 风险 | 应对 |
|------|------|
| Creem Discount API 有 bug | 备选: 多产品映射方案（为每个折扣创建独立 Creem 产品） |
| JSON 文件过大（>10万条） | 迁移到 Cloudflare D1 或 SQLite |
| 博主虚假转化 | 人工审核 + 异常模式监控（同IP批量购买等） |
| Creem metadata 丢失 | 后端通过 discount_code 反查 promo_codes 表 |
