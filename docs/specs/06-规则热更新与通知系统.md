# 06 - 规则热更新与通知系统

更新日期: 2026-03-12  
状态: 生效中

## 1. 规则更新分层

1. 免费版：手动同步，最短间隔 7 天。
2. 试用/付费：自动热更新（高频巡检）。
3. 所有策略以后端命令为准，前端仅做提示与触发。

## 2. 命令与状态

1. `get_rule_update_status`
2. `check_rule_update`
3. `download_and_apply_rules`

## 3. 安全约束

1. 免费策略节流在后端生效，不能通过前端或本地存储绕过。
2. 规则来源和版本必须可见（内置/远端）。
3. 更新结果必须落通知与事件记录。

## 4. 通知系统

支持：

1. 持久化通知列表
2. 未读计数
3. 安全/系统/更新分类
4. 桌面通知联动

## 5. 验收标准

1. 同步成功后可见版本变化。
2. 免费版在 7 天窗口内触发同步会被后端拒绝并给出下次可用时间。
3. 自动热更新不影响主流程可用性。
