# 05 - 支付与许可证系统

更新日期: 2026-03-12  
状态: 最小闭环可用（生产闭环持续建设）

## 1. 业务模型

1. 14 天全功能试用。
2. 激活码升级（30 天 / 365 天 / 永久）。
3. 免费版保留可用主流程，但自动化能力降级为手动。

## 2. 客户端许可证

命令：

1. `start_trial`
2. `activate_license`
3. `check_license_status`
4. `deactivate_license`

## 3. 安全约束

1. 激活码采用 `AGSH.<payload>.<signature>` 签名。
2. 许可证状态每次读取都后端复验签名与试用锁，防本地篡改绕过。
3. 功能门禁以后端为准，不允许仅前端控制。

## 4. License Gateway（最小运营）

已支持：

1. webhook 验签
2. 幂等发码
3. 重发码
4. 撤销码
5. 基础审计日志

## 5. 仍需补强

1. 生产级高可用与对账自动化。
2. 撤销状态与客户端周期同步增强。
3. 运维告警与失败重试策略强化。
