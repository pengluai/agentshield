# Creem 开卖前最终状态与激活码购买流程（2026-03-17）

## 当前后台状态判断
基于用户提供的 Creem 截图：
- 页面顶部显示：`1 需要设置 / 1 已验证 / 0 已准备好支付 / 1 家门店`
- 门店卡片显示：
  - `身份验证`
  - `需要支付`
  - 按钮：`设置验证`

这说明：
1. 至少有一项验证已经通过
2. 但当前门店还没有进入 `ready to accept payments` 状态
3. 也就是说：**还不能正式销售**

结合 Creem 官方文档：
- 完整上线链路是：
  `Business Details -> KYC/KYB Verification -> Payout Account Setup -> Account Review by Creem Team -> Live Payments Enabled`
- 每个 store 都需要单独完成 payout account 设置
- 即便你已经有通过审核的钱包/银行账户，仍然需要把它配置到当前 store 上

因此，当前最可能还差的就是：
- 把已验证的 payout account / crypto wallet **链接到当前 store**
- 等该 store 状态从 `需要支付` 变成 `已准备好支付`

## 你还需要完成什么才能正式销售
### 后台侧
1. 点击 `设置验证`
2. 选择当前已经通过审核的 payout account（银行 / Alipay / Crypto wallet）
3. 把它绑定到当前 store
4. 确认页面状态变成：
   - `已准备好支付 = 1`
   - 当前 store 不再显示 `需要支付`
5. 确认当前是 `Live / Production` 环境，而不是 Test Mode

### 项目侧
当前官网仍然是审核保护模式：
- 月付 / 年付 / 终身购买按钮都还是 disabled
- 文案仍写着审核完成后才开放支付

因此，正式开卖前还必须完成：
1. 把 storefront 的付费按钮从 disabled 状态切回 live checkout 链接
2. 用真实 live checkout URL 替换按钮目标
3. 再做一轮真实购买验收

## 用户如何购买、支付、拿到激活码
当前项目代码已经实现的真实链路如下：

### 1. 用户购买
用户在官网点击：
- 30 天激活码
- 365 天激活码
- 终身激活码

按钮跳转到对应的 Creem checkout link。

### 2. 用户支付
用户在 Creem checkout 完成支付。
支付方式由 Creem 前台可用方式决定（卡、Apple Pay、Google Pay、PayPal 等视地区而定；商家提现方式是另一回事）。

### 3. Creem 发送 webhook
支付成功后，Creem 向你的 license gateway 发送 `checkout.completed` webhook。
如果是订阅续费，则发送 `subscription.paid`。
如果退款，则发送 `refund.created`。

### 4. license gateway 签发激活码
license gateway 收到 webhook 后：
- 创建 order 记录
- 生成/签名 activation code
- 记录 license
- 保存到状态存储

### 5. 邮件发码
gateway 使用 Resend 向用户下单邮箱发送激活码邮件。
邮件中包含：
- plan
- expires_at
- activation code

### 6. 用户在 App 中激活
用户打开 AgentShield App
-> 输入激活码
-> 调用 `activate_license`
-> 本地校验签名并保存 license 信息

### 7. 后续在线校验
App 后续调用 `/client/licenses/verify`：
- 确认 license 是否仍有效
- 处理退款、撤销、续费等状态变化

## 现在的最终结论
### 你还差什么
还差两件事：
1. **Creem store 级别的 payment setup 完成**
2. **把官网购买按钮切回 live**

### 一旦这两件事完成
你就可以正式销售，用户路径就是：
`官网购买 -> Creem 支付 -> webhook -> 邮件收到激活码 -> App 粘贴激活`
