# AgentShield 公开售卖倒排与打包执行手册（2026-03-13）

## 1. 目标
在不改内核定位（仅覆盖 AI 工具生态安全治理）的前提下，把项目从“试点分发”推进到“可公开售卖”。

本手册只包含仓库可执行项与可验收门禁，避免“口头上线”。

## 2. Go/No-Go 硬门禁
公开售卖前必须全部通过：

1. `pnpm run release:public:ready` 通过。
2. 法务文档齐全并已由法务最终审校：
   - `docs/legal/privacy-policy.md`
   - `docs/legal/terms-of-service.md`
   - `docs/legal/eula.md`
   - `docs/legal/refund-policy.md`
3. 许可证闭环通过演练：
   - `order_created` 可发码
   - `order_refunded` 或 `subscription_payment_refunded` 可撤销 license
4. `src-tauri/tauri.release.json` 使用真实值（非占位符）且启用 updater artifacts。
5. 证书与签名密钥已注入发布环境（不写入仓库）。

## 3. 倒排节奏（D-7 到 D-Day）

### D-7 ~ D-5：基础闭环
1. 完成公开售卖门禁脚本与打包脚本。
2. 完成法务文档草案并进入法务审校。
3. 完成退款撤销许可证联调。

### D-4 ~ D-3：灰度演练
1. 在测试商店演练购买、发码、激活、退款、撤销全链路。
2. 检查 webhook 重放、重复事件、异常签名处理。
3. 对外 FAQ 与客服 SOP 定稿。

### D-2 ~ D-1：预发布冻结
1. 代码冻结（仅允许阻断级修复）。
2. 执行 `release:public:ready` 全套门禁。
3. 产出候选包并做实机安装回归。

### D-Day：正式发布
1. 执行 `pnpm run release:public:bundle`。
2. 上传签名产物与更新清单。
3. 发布公告并开启订单监控与客服值守。

## 4. 关键命令

```bash
# 1) 公开售卖前置检查（含测试与签名前提）
pnpm run release:public:ready

# 2) 公开售卖正式打包（ready 通过后执行）
pnpm run release:public:bundle

# 3) 本地 License Gateway 联调
set -a && source .env.license-gateway.local && set +a
pnpm run gateway:dev
```

## 5. 角色分工（最小配置）
1. 发布负责人：执行门禁、打包、产物发布。
2. 后端负责人：支付回调、license 签发/撤销、运营后台。
3. 客服负责人：退款工单、激活失败工单、升级咨询。
4. 法务负责人：隐私、条款、EULA、退款政策终审。

## 6. 验收标准
1. 新用户 5 分钟内可完成下载、安装、激活、首次扫描。
2. 退款用户在 webhook 到达后，关联 license 自动撤销。
3. 没有密钥缺失、占位符配置、或法务文档缺失即可触发打包的风险路径。
4. 回滚策略明确：发布后 30 分钟内可回滚到上一稳定版本。

## 7. 风险与兜底
1. 证书/签名缺失：阻断打包，不允许“先发后补签”。
2. webhook 不稳定：保留人工 revoke/reissue 管理口，且记录审计日志。
3. 法务未完成：No-Go，禁止公开售卖。
4. 支付渠道变更：先在测试环境跑完整演练再切生产。
