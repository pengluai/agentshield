# 14 - 干净机实机回归清单

更新日期: 2026-03-09  
状态: 待执行  
适用范围: AgentShield `Windows 11` / `macOS 14+`

## 1. 目的

本清单用于把当前仓库的“试点可用”推进到“可公开交付”前的最后实机验收。

目标不是泛泛“点一遍页面”，而是验证这三件事：

- 每个关键按钮都绑定真实后端能力
- 运行时守卫能对真实 MCP / Skill 做监控与处置
- 安装、升级、卸载、隔离、通知在干净环境中不会失效

## 2. 测试前置

每台机器执行前，确认：

- 新用户或干净系统环境
- 未预装历史 AgentShield 配置
- `OpenClaw` 可安装
- 至少准备 1 个安全的本地测试 MCP
- 至少准备 1 个安全的本地测试 Skill
- 至少准备 1 个“会外联”的受限测试组件

建议每台机器记录：

- OS 版本
- CPU 架构
- Node 版本
- 是否已安装 Xcode / Visual Studio Build Tools
- 代码签名状态

## 3. 回归步骤

### A. 首启与权限

1. 启动 AgentShield
2. 完成 onboarding
3. 验证通知权限按钮触发真实系统授权
4. 验证磁盘 / 辅助功能 / Automation 入口跳到真实系统设置

### B. OpenClaw 与商店

1. 安装 OpenClaw
2. 打开 OpenClaw 专区
3. 安装一个商店 MCP
4. 验证该组件出现在“已安装管理”
5. 验证版本、来源、运行时信任状态显示正确

### C. 手动新增 MCP / Skill

1. 在目标 AI 工具配置目录中手动添加一个 MCP
2. 在 Skills 目录中手动加入一个 Skill
3. 返回 AgentShield
4. 验证两者都被发现并进入运行时守卫 registry
5. 验证手动新增组件默认为 `待审批`

### D. 受控启动

1. 将测试 MCP 审批为 `受限`
2. 点击“受控启动”
3. 验证活动会话中出现真实 PID
4. 将测试 Skill 审批为 `受限`
5. 点击“受控启动”
6. 验证 Skill 成功启动，或在缺少入口时收到明确拒绝信息

### E. 网络与自动处置

1. 对 `restricted` 组件配置 allowlist
2. 触发已知允许外联，验证不被误杀
3. 触发未知外联，验证自动终止并隔离
4. 对 `unknown + manual_*` 组件触发外联，验证自动终止
5. 验证“近期处置事件”显示真实审计记录

### F. 升级 / 卸载 / 定位文件

1. 检查更新
2. 对托管安装组件执行升级
3. 卸载组件
4. 验证真实配置或 Skill 目录被移除
5. 验证定位文件按钮打开真实本地路径

## 4. 记录模板

每个用例至少记录：

- 机器
- 组件名
- 来源
- trust_state
- network_mode
- 操作步骤
- 实际结果
- 是否通过
- 是否存在误杀 / 漏报
- 相关截图或日志路径

## 5. 放行标准

只有同时满足以下条件才允许标记为“可公开交付”：

- Win/mac 两个平台关键用例全部通过
- 无阻断级 UI 假按钮
- 无高优先级崩溃
- 误杀案例可解释且有可接受 workaround
- 签名 / notarization / updater 链已真实可用
