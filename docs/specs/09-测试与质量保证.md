# 09 - 测试与质量保证

更新日期: 2026-03-12  
状态: 生效中

## 1. Repo-local 质量门

必须持续通过：

1. `pnpm typecheck`
2. `pnpm lint`
3. `pnpm test`
4. `cargo test --manifest-path src-tauri/Cargo.toml`
5. `pnpm exec playwright test e2e/smoke.spec.ts --reporter=line`
6. `RELEASE_PROFILE=pilot pnpm run release:gate`

## 2. 稳定性验证

1. `pnpm run test:soak`
2. `pnpm run report:guard`

## 3. 质量红线

1. 不允许假按钮、假进度、演示态替代真实执行。
2. 不允许仅前端门禁，后端必须有强约束。
3. 不允许泄露敏感数据到日志或 UI 明文。

## 4. 发布分层

1. `pilot`：允许 ad-hoc 签名分发试点包。
2. `public`：必须完成签名、公证、更新密钥与干净机回归。

## 5. 仍需人工验收

1. Win/mac 干净机全流程回归（见 `14-干净机实机回归清单.md`）。
2. 生态实机覆盖扩展（第三方宿主动作级信号）。
