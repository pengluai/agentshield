# 21 - 商用发布前复核与阻塞项更新

更新日期: 2026-03-11  
适用范围: AgentShield `macOS` / `Windows` / GitHub 分发桌面版  
当前结论: **Pilot 可继续，面向零基础付费公开售卖仍是 No-Go**

## 1. 本轮结论

这轮不是重新做一份泛化审查，而是把 2026-03-11 仓库现状、外部最新 OpenClaw / MCP 风险、以及本轮实际落地的修复重新对齐。

当前可以确认的事实：

1. 仓库内 `release:gate` 已重新跑通：
   - `cargo test`
   - `pnpm build`
   - `pnpm test`
   - `pnpm exec playwright test e2e/smoke.spec.ts --reporter=line`
   - `pnpm tauri info`
2. Windows 宿主发现补丁仍然成立，相关 Rust 测试通过。
3. 浏览器壳关键 smoke 已通过，之前“Scan 后仍落入通用失败态”的问题已不再复现。
4. 我这轮新增了后端强制验票：高危动作不再只是前端先弹窗、后端直接执行。

但仍然不能宣称：

- “整个项目已经可以面向零基础用户安全售卖”；
- “已经能阻止 OpenClaw / MCP / Skill 在任何路径下未经授权删文件、删邮件、导出敏感信息”；
- “月付 / 年付 / 激活码 / 撤销 / 续费已经形成真实商用闭环”。

## 2. 本轮已落地修复

### 2.1 高危动作从“前端审批”升级为“后端也必须验票”

已接入真实后端验票的链路：

- OpenClaw 卸载
- 已安装 MCP / Skill 卸载
- 密钥明文导出 / 复制
- 密钥删除
- 一键修复批量收紧配置文件权限

实现方式：

- `runtime_guard` 新增一次性执行票据；
- 用户先在审批中心点头；
- 后端命令执行前必须校验票据和动作作用域；
- 没票或票据不匹配时，后端直接拒绝执行。

这修复了一个很关键的真实性问题：  
此前即使前端弹出审批，只要有人绕过 UI 直接调用后端命令，危险动作仍可执行。现在这几条链路已经收紧。

### 2.2 Repo-local 发布门重新验证

本轮重新执行后，`pilot` 配置下的 repo-local gate 已通过。  
仍然存在的提示属于已知外部前置条件：

- `Xcode` 未安装
- `APPLE_SIGNING_IDENTITY` 未配置

这两项只阻塞公证/公开发布，不阻塞当前 GitHub `pilot` 分发链路。

## 3. 外部最新风险核实

### 3.1 OpenClaw 本地代理被恶意网站接管

最新公开证据已经确认这不是理论风险：

- Oasis Security 在 2026 年 3 月披露 `ClawJacked`，说明任意网站都可能通过本地 WebSocket 接管 OpenClaw。
- OpenClaw 官方在 2026-03-01 的说明中确认：
  - 高危漏洞存在；
  - 已在 `v2026.2.25` 中修复；
  - 修复项包括认证限速、新设备显式审批、加强本地 WebSocket 安全。

对 AgentShield 的直接意义：

- 必须继续把 OpenClaw 版本检测、`openclaw security audit`、本地风险解释、升级提示放在 P0；
- 必须让用户看到“为什么要升级”和“升级后解决什么”；
- 不能把“localhost”默认视为安全边界。

### 3.2 MCP 官方仍明确强调 SSRF / Session Hijack / Local Server Compromise

MCP 官方安全最佳实践当前仍明确强调：

- SSRF
- session hijacking
- local MCP server compromise
- 最小权限 / scope minimization
- 禁止 token passthrough

对 AgentShield 的意义：

- 你的产品聚焦点仍然是对的，不能泛化成“系统杀毒”；
- 应继续聚焦：
  - 本机 MCP / Skill 来源和落点发现
  - 高危命令和外联解释
  - allowlist / 最小权限
  - 本地 server / localhost / loopback 风险

### 3.3 Skill 供应链风险仍在扩大

2026 年 2 月以来，公开研究仍持续指向 ClawHub / skill 供应链被恶意滥用：

- Snyk 披露 `clawdhub1` 等伪装技能在市场中投放恶意载荷；
- The Hacker News 汇总了更多恶意 skill、AMOS 分发、以及与加密诈骗相关的恶意条目；
- 这意味着“只显示已安装列表”远远不够，必须继续做：
  - 来源透明
  - 高风险命令模式识别
  - 可疑下载链接 / 外链 / base64 执行链提示
  - `SKILL.md` 与脚本入口的联合扫描

## 4. 仍然阻塞公开付费发售的真实问题

### 4.1 运行时守卫仍未形成“外部 agent 动作级拦截平面”

当前 `runtime_guard` 的后端真实强项仍然主要是：

- 组件启动审批
- 会话发现
- 外联观测
- allowlist / kill / quarantine

但对你真正想卖的承诺来说，核心缺口还在：

- 外部 OpenClaw / MCP / Skill 真正执行 `file_delete`
- `bulk_file_modify`
- `email_send`
- `email_delete_or_archive`
- `browser_submit`
- `payment_submit`

这些动作还没有形成“系统外部 agent 一发起就能被 AgentShield 精确预览并拦住”的统一后端控制面。  
本轮修掉的是 AgentShield 自己 UI 发起的高危动作，不是所有第三方 agent 的宿主机行为。

### 4.2 商业化许可证闭环仍不成立

当前许可证实现虽然已经优于“纯本地明文文件 + 30 天试用”的旧状态，但仍不足以支撑正式订阅收费：

- 仍无在线 seat / 续费 / 撤销 / 停用同步
- 仍无真实月付 / 年付状态回传
- 仍无 license service
- 仍无法做到“付款取消后自动失效”

如果你现在就开始对外卖，最现实的边界仍然是：

- 试用
- 激活码
- 手动运营

而不是自动化月付 / 年付 SaaS 订阅闭环。

### 4.3 真机验收仍未完成

仓库内 gate 通过不等于 Win/mac 真机都可商用：

- 干净 Windows 11 真机
- 干净 macOS 14+ 真机
- 首启权限链
- OpenClaw 安装 / 更新 / 卸载
- 手动新增 Skill / MCP
- 运行时守卫误杀 / 漏报

这些仍需要按 `14-干净机实机回归清单.md` 执行。

## 5. 当前可对外说法

可以说：

> AgentShield 已具备 OpenClaw / MCP / Skill 生态的本机发现、静态扫描、受控启动、运行时联网审计与部分高危动作后端验票能力，适合 GitHub 试点分发与技术用户试用。

不能说：

> AgentShield 已经完整阻止所有 OpenClaw / MCP / Skill 的未授权危险动作，适合零基础用户无门槛付费使用。

## 6. 下一步优先级

P0:

1. 把邮件类、浏览器提交类、支付类动作接入同样的后端验票。
2. 设计最小可运营 license service：
   - 激活码校验
   - 14 天试用
   - 月付 / 年付状态拉取
   - 撤销 / 停用
3. 做 Win/mac 干净机实机回归。

P1:

1. 把 OpenClaw 风险解释做成用户可见卡片：
   - 当前版本
   - 官方修复版本
   - 本机是否存在已知高危配置
2. 把恶意 Skill / MCP 的来源、下载链、可疑命令片段解释得更直白。
