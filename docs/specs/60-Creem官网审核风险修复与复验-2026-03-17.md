# Creem 官网审核风险修复与复验

日期：2026-03-17

## 背景
针对 AgentShield 官网可能影响 Creem account review 的内容风险，按官方文档与线上实测结果进行了整改与复验。

## 官方依据
1. Creem 全文档：<https://docs.creem.io/llms-full.txt>
2. Creem 联系支持：<https://www.creem.io/contact>
3. Cloudflare Workers Static Assets：<https://developers.cloudflare.com/workers/static-assets/>
4. Cloudflare Wrangler 配置：<https://developers.cloudflare.com/workers/wrangler/configuration>

关键信息：
- Support email 必须是真实、可联系，并与网站展示一致。
- Account review 最常见问题包括：support email mismatch、website not accessible、missing legal pages、false information、product not ready。
- KYC 被拒后，官方建议携带 Store ID 联系支持请求 reset 或解释具体原因。

## 本轮修复内容
### 1. 去掉会进入报错页的支付入口
- 年付、终身、月付购买入口全部改为 `审核中 / coming soon` 的禁用状态。
- 保留价格展示，但不再把审核员带到 `Live payments are not enabled` 的 checkout 报错页。

### 2. 收紧不可证实营销文案
已移除或软化：
- `全球首款`
- `The First Desktop Security Suite for AI Tools`
- `3,900万+ / 39M+ API keys`
- `10,000+ developers`
- `92%`

### 3. 删除具名 testimonials 风险
- 去掉具名用户评价、头像、身份头衔。
- 改为中性的适用场景卡片：独立开发、团队试点、安全复核。

### 4. FAQ 与法律页对齐
- 本地运行表述改为：核心扫描主要本地进行，但许可证校验、规则更新、购买流程可能联机。
- 退款 FAQ 改为与 Refund Policy 一致的 14 天默认窗口与适用条件。

### 5. 法律页入口更显性
Footer 现在直接露出：
- Privacy Policy
- Terms
- Refund Policy
- EULA
- GitHub
- Contact

### 6. 页面完整性补充
- 新增内联 favicon，消除线上控制台的 `favicon.ico` 404。

## 当前剩余风险
1. Creem 商家收款审核本身尚未通过，因此无法恢复真实支付按钮。
2. KYC / SUMSUB 记录当前仍是 `Rejected`，这部分需要按官方建议联系 Creem support 处理。
3. 官网虽然已更符合 review 口径，但不能替代 Creem 后台的实名与 payout 开通。

## 结论
就官网内容本身而言，本轮已把最容易触发 Creem review 的前台问题基本收干净：
- 不再有 broken checkout
- 不再有明显不可证实的用户量/评价/绝对化宣传
- 法律页可见且可访问
- 联系邮箱一致

下一步应优先处理 Creem 后台的 KYC rejected 工单，而不是继续改官网文案。
