# 01 - 基础框架与设计系统

更新日期: 2026-03-12  
状态: 与当前代码同步

## 1. 技术栈

1. 桌面框架：Tauri v2
2. 前端：React 18 + Vite + TypeScript + Zustand
3. 样式：Tailwind + 组件化样式封装
4. 测试：Vitest + Playwright + Rust 单测

## 2. 设计原则

1. 极简主路径：打开即扫描/管理，不强依赖新手教学流程。
2. 风险优先：高风险动作默认拦截，用户显式确认后执行。
3. 诚实降级：浏览器壳中桌面专属能力必须给出可理解提示。

## 3. UI 组件约束

1. 审批交互采用右下角轻量卡片，支持“查看详情”展开技术字段。
2. 免费/付费差异必须在关键动作节点显式提示，而非隐藏在定价页。
3. 颜色语义统一：
   - 绿色：安全完成
   - 橙/黄：风险提醒
   - 红色：阻断或失败

## 4. 国际化与文案

1. 文案统一从 `src/constants/i18n.ts` 读取。
2. 主流程文案优先“后果描述”，术语置于详情层。
3. 时间敏感文案必须标注明确频率（例如 7 天规则同步）。
