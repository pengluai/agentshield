## 2. 当前推荐的发布路径

### 路径 A：保留源码仓库私有，单独建立公开下载仓库

这是当前最稳妥的方式，因为：

1. 你的源码仓库现在是私有的。
2. 私有仓库不适合作为面向大众的公开下载入口。
3. 单独建立一个公开“下载仓库”不会暴露源码，但可以公开 release、README、安装说明和更新记录。

推荐做法：

1. `pengluai/agentshield` 继续保持私有，作为研发仓库。
2. 新建一个公开仓库，例如：
   - `pengluai/agentshield-downloads`
3. 公开仓库里只放：
   - README
   - Release notes
   - DMG / EXE 安装包
   - 校验值
   - 常见问题

### 路径 B：直接把当前源码仓库改成公开

优点：

1. 维护简单
2. 下载与源码在一个仓库

缺点：

1. 源码直接公开
2. 后续商用策略、风控规则、实现细节都更容易被观察

AgentShield 是一个桌面安全助手，专门帮你检查本机 AI 工具、MCP / Skill 配置、密钥暴露、危险自动化和高风险执行行为。

### 适合谁

1. 同时安装了多个 AI 客户端、插件、MCP、Skill 的用户
2. 想知道本地 AI 工具有没有“越权”“偷跑”“泄露密钥”的用户
3. 想先看风险，再决定是否信任某个 AI 工作流的创作者、开发者和普通用户

### 它解决什么问题

1. **看不见风险**：不知道哪个 AI 工具装了哪些 MCP / Skill
2. **怕泄露密钥**：担心 API Key、Token、凭证散落在配置里
3. **怕误删文件**：担心危险脚本、自动化、批处理被误触发
4. **怕后台乱跑**：担心 AI 工具在后台偷偷执行高风险操作

### 当前版本能做什么

1. 扫描本地 AI 工具和 MCP / Skill 配置
2. 检查权限失控、密钥暴露、隐私泄漏、恶意插件和后台偷跑风险
3. 显示 MCP / Skill 属于哪个 AI 工具
4. 提供系统钥匙串级别的密钥保险库
5. 在执行删除敏感信息、批量修复、卸载敏感组件前要求确认
6. 支持试用与激活码升级

### 下载提示

1. macOS：下载 `DMG`
2. Windows：下载 `x64-setup.exe`
3. 首次安装如遇到系统拦截，请在系统安全设置中允许一次

## 5. 你如何拿到激活码

### 5.1 当前最快方式：本地签发

仓库已经提供封装脚本：

```bash
cd /Users/luheng/Downloads/ai01/agentshield
pnpm run license:issue:public -- --billing-cycle monthly --customer buyer@example.com --days 31
```

其他示例：

```bash
pnpm run license:issue:public -- --billing-cycle yearly --customer buyer@example.com --days 366
pnpm run license:issue:public -- --billing-cycle lifetime --customer buyer@example.com
```

### 5.2 发码前提

1. `.env.public-sale.local` 中已经有：
   - `AGENTSHIELD_LICENSE_SIGNING_SEED`
   - `AGENTSHIELD_LICENSE_PUBLIC_KEY`
2. 当前本地 seed 与 public key 已匹配，可直接签发有效激活码。

### 5.3 建议的运营动作

1. 收款成功后再签发
2. 每个客户一单一码
3. 在你自己的订单表中记录：
   - 客户邮箱
   - 方案（月付 / 年付 / 终身）
   - 签发时间
   - 过期时间
   - 激活码对应订单号
