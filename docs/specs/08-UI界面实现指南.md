# 08 - UI 界面实现指南

更新日期: 2026-03-12  
状态: 生效中

## 1. 页面结构

1. 首页（智能守护）
2. 安全扫描
3. 技能商店
4. 已安装管理
5. 密钥保险库
6. OpenClaw 管理
7. 通知中心
8. 设置
9. 升级完整版

## 2. 关键交互准则

1. 审批入口：右下角轻量卡片，不做全屏强打断。
2. 主流程文案：先说后果，再给按钮；术语放“查看详情”。
3. 免费/付费分层：在动作按钮处显式提示，不隐藏在说明页。
4. 浏览器壳降级：桌面专属动作必须诚实提示。

## 3. 错误与反馈

1. 所有操作必须返回真实成功/失败状态。
2. 失败提示要包含用户可执行的下一步。
3. 系统设置跳转失败时提供手动路径与复检入口。

## 4. 交付门槛

1. 无演示态按钮。
2. 无“点击有动画但无后端执行”的伪成功。
3. 关键路径（扫描、修复、安装、卸载、审批）有自动化测试覆盖。
